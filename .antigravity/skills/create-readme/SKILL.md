# README Creator

## Task
1. **Review**: Analyze the entire workspace and project structure.
2. **Structure**: Focus on Features, Tech Stack, Setup, and Usage.
3. **Tone**: Informative, appealing, and concise.
4. **Style**: Use GitHub Flavored Markdown (GFM) and admonitions for notes/tips.

## Guidelines
- Avoid overusing emojis.
- Exclude LICENSE/CONTRIBUTING if files already exist.
- Use project logos if available.
- Link to relevant documentation and local artifacts.
