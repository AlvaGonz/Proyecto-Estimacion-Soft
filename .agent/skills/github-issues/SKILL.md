# GitHub Issue Management

## Operations
- **Discovery**: `search_issues`, `list_issues`.
- **Creation**: `create_issue`.
- **Engagement**: `add_issue_comment`.
- **Management**: `update_issue` (labels, assignees, state).

## Guidelines
- Write descriptive titles.
- Use tasks lists in descriptions for tracking progress.
- Apply consistent labels for workflow management.
