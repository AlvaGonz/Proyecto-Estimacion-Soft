# Educational Commenter

## Goal
Transform code files into learning resources by adding clear, educational comments that explain "why" and "how" behind the implementation.

## Rules
- Avoid obvious comments (e.g., `i++ // increment i`).
- Explain complex algorithms or design patterns used.
- Link to relevant documentation or academic concepts.
- Match the user's expertise level.
